        return blocks[Index(x, y, z)];
    }

    void SetBlock(int x, int y, int z, const Block& block) {
        blocks[Index(x, y, z)] = block;
        meshDirty = true;
    }
};

Ah yes — the gaps you’re seeing are classic block-based rendering artifacts. Let me explain exactly why this happens and how to fix it:

1. Why Gaps Happen
Even though your blocks are in a grid, they often appear separated because of floating point precision, mesh scaling, or individual cube meshes not perfectly aligned.

Common causes:

Each block is rendered as its own mesh → faces that touch each other don’t merge → tiny gaps appear.

UV padding / texture bleeding → edges of textures can create visual gaps.

Floating point errors in world positions → especially if your ship is far from origin.

2. Solutions
A. Use a Strict Grid System
Yes! A grid is essential:

Define all block positions as integers (x, y, z) in block units, not world units.

Multiply by block size only when generating vertex positions:

Vector3 worldPos = Vector3(gridX, gridY, gridZ) * BLOCK_SIZE;
BLOCK_SIZE = 1.0f usually works best.

Avoid adding offsets manually; keep every block aligned on the same grid.

B. Merge Adjacent Faces (Greedy Mesh)
Remove internal faces (faces between two solid blocks).

This prevents the “see-through gaps” caused by tiny seams.

This also dramatically reduces vertex count and improves performance.

C. Avoid Scaling Blocks
Don’t scale cubes to, say, 0.99 or 1.01 — this will create visible gaps.

Always use exact block size and precise grid positioning.

D. Slight Overlap / Expansion Trick (Optional)
If tiny gaps still appear due to floating point precision, slightly expand cubes:

VERTEX_POSITION = basePos * BLOCK_SIZE * 1.001f;  // 0.1% bigger
Tiny, barely visible, but closes gaps without noticeable distortion.

This is often done in voxel engines like Minecraft clones.

E. Use Chunked Rendering
Render chunks of blocks as one mesh instead of per-block.

Internal faces are never drawn, so only outer surfaces are visible.

This makes ships appear solid with no gaps.

✅ Recommended Approach for Your Ship Builder
Keep all blocks on a strict integer grid.

Use chunked greedy meshing to combine faces.

Remove internal faces entirely.

Assign textures carefully with padding to prevent UV bleeding.

Optional: Slight block expansion if floating point gaps persist.

7. Gap-Free Face Builder
VoxelMesh.cpp

#include "VoxelMesh.h"
#include "Chunk.h"

const float BLOCK_SIZE = 1.0f;

static bool IsAir(Chunk& chunk,int x,int y,int z)
{
    if(x<0||y<0||z<0) return true;
    if(x>=CHUNK_SIZE||y>=CHUNK_SIZE||z>=CHUNK_SIZE) return true;

    return chunk.blocks[x][y][z].type == BlockType::Empty;
}

static void AddFace(
    Mesh& mesh,
    float x,float y,float z,
    float nx,float ny,float nz)
{
    uint32_t start = mesh.vertices.size();

    Vertex v1{ x,y,z,nx,ny,nz,0,0 };
    Vertex v2{ x+1,y,z,nx,ny,nz,1,0 };
    Vertex v3{ x+1,y+1,z,nx,ny,nz,1,1 };
    Vertex v4{ x,y+1,z,nx,ny,nz,0,1 };

    mesh.vertices.push_back(v1);
    mesh.vertices.push_back(v2);
    mesh.vertices.push_back(v3);
    mesh.vertices.push_back(v4);

    mesh.indices.push_back(start+0);
    mesh.indices.push_back(start+1);
    mesh.indices.push_back(start+2);

    mesh.indices.push_back(start+2);
    mesh.indices.push_back(start+3);
    mesh.indices.push_back(start+0);
}

Mesh BuildChunkMesh(Chunk& chunk)
{
    Mesh mesh;

    for(int x=0;x<CHUNK_SIZE;x++)
    for(int y=0;y<CHUNK_SIZE;y++)
    for(int z=0;z<CHUNK_SIZE;z++)
    {
        Block& b = chunk.blocks[x][y][z];

        if(b.type == BlockType::Empty)
            continue;

        float px = x * BLOCK_SIZE;
        float py = y * BLOCK_SIZE;
        float pz = z * BLOCK_SIZE;

        if(IsAir(chunk,x+1,y,z))
            AddFace(mesh,px+1,py,pz,1,0,0);

        if(IsAir(chunk,x-1,y,z))
            AddFace(mesh,px,py,pz,-1,0,0);

        if(IsAir(chunk,x,y+1,z))
            AddFace(mesh,px,py+1,pz,0,1,0);

        if(IsAir(chunk,x,y-1,z))
            AddFace(mesh,px,py,pz,0,-1,0);

        if(IsAir(chunk,x,y,z+1))
            AddFace(mesh,px,py,pz+1,0,0,1);

        if(IsAir(chunk,x,y,z-1))
            AddFace(mesh,px,py,pz,0,0,-1);
    }

    return mesh;
}

Compute Ship Mass + Center of Mass
void ComputeMass(Ship& ship)
{
    float totalMass = 0.0f;
    Vector3 weightedPos = {0,0,0};

    for(auto& block : ship.blocks)
    {
        auto props = BlockPhysicsTable[block.type];

        totalMass += props.mass;
        weightedPos += block.worldPos * props.mass;
    }

    ship.mass = totalMass;
    ship.centerOfMass = weightedPos / totalMass;
}

2. Ship Data Container
Each ship owns a 3D block grid.

#include <unordered_map>

struct GridHash
{
    size_t operator()(const glm::ivec3& v) const
    {
        return std::hash<int>()(v.x * 73856093 ^ v.y * 19349663 ^ v.z * 83492791);
    }
};

class ShipGrid
{
public:

    float gridSize = 1.0f;

    std::unordered_map<glm::ivec3, ShipBlock, GridHash> blocks;

    bool HasBlock(glm::ivec3 pos)
    {
        return blocks.find(pos) != blocks.end();
    }

    void AddBlock(const ShipBlock& block)
    {
        blocks[block.gridPos] = block;
    }

    void RemoveBlock(glm::ivec3 pos)
    {
        blocks.erase(pos);
    }
};

9. Physics Center of Mass Solver
Ships must rotate realistically.

Each block contributes mass.

Center of Mass
COM = Σ(mass * position) / totalMass
Implementation
glm::vec3 ComputeCenterOfMass(ShipVoxelGrid& grid)
{
    float totalMass = 0;
    glm::vec3 weightedSum(0);

    for (auto& chunkPair : grid.chunks)
    {
        auto& chunk = chunkPair.second;

        for(int x=0;x<CHUNK_SIZE;x++)
        for(int y=0;y<CHUNK_SIZE;y++)
        for(int z=0;z<CHUNK_SIZE;z++)
        {
            Block& b = chunk.blocks[x][y][z];

            if(!b.active) continue;

            glm::vec3 pos = glm::vec3(x,y,z);

            weightedSum += pos * b.mass;
            totalMass += b.mass;
        }
    }

    return weightedSum / totalMass;
}
10. Thruster Torque Solver
Thrusters rotate ship depending on distance from COM.

Torque = r × F
Implementation
glm::vec3 ComputeThrusterTorque(glm::vec3 thrusterPos,
                                glm::vec3 thrustDir,
                                float thrust,
                                glm::vec3 centerOfMass)
{
    glm::vec3 r = thrusterPos - centerOfMass;

    glm::vec3 force = thrustDir * thrust;

    return glm::cross(r, force);
}

Emergent Survival Mechanics: Manage oxygen, food, water, temperature, sleep, and stress. Life support, fabrication, and ship systems must be maintained for both yourself and your crew.

Modular Ship & Station Construction: Fully voxel-based building. Build, repair, and upgrade your first ship to explore space and gather vital resources.

Multi-Mode Universal Tool: Start with a procedural tool that evolves visually and functionally, capable of resource extraction, defense, and utility interactions. Battery management and heat dynamics create meaningful tactical decisions.

Procedural Crew & NPCs: Awaken alongside a procedurally generated crew with unique skills, backstories, and personalities. Assign tasks, manage morale, and survive together.

Nested Inventory System: Each item and container has volume, weight, and sub-containers, enabling deep tactical inventory management.

Emergent Storytelling: Procedurally generated logs, faction data, fractured ships, and escape pods reveal a fractured history of space stations, factions, and lost technology. Every discovery shapes your survival and emergent narrative.

Dynamic Station Restoration: Systems brought online progressively improve survival, unlock new crafting and shipbuilding capabilities, and reveal story threads across the universe.

Gameplay Loop:

Wake in a derelict pod → assess and restore life support, power, and station systems.

Unlock survival tools, EVA capabilities, and procedural blueprints.

Build your first ship and EVA to collect resources and salvage.

Explore nearby debris, fractured ships, and escape pods for rare resources, secrets, and faction tech.

Expand your survival footprint to planets, asteroid bases, or new stations — and uncover the procedural mysteries of the universe.

Vision:
Cryovast is a player-driven, procedural sandbox where every repair, exploration, and resource collected impacts survival, story, and progression. It’s a space survival experience with emergent narrative depth, rewarding creativity, tactical thinking, and exploration.
